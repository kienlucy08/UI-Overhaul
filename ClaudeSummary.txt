## 2. GuyFacilitiesQCPage.tsx — Full Rebuild

### Architecture
Mirrors QCEditorPage shell exactly: toolbar → sidebar → content area → footer nav.

### Sections (sidebar + content)
  - Guy Compounds
  - Guy Photos
  - Deficiencies
  - Catch All

### Guy Compounds — Drill-in pattern (not accordion)
- Compounds section shows a responsive grid of nav cards (1/2/3 cols by breakpoint).
- Clicking a card drills into that compound's own "page" (activeCompoundId state).
- Breadcrumb in sticky header: "Guy Compounds › Compound B" — clicking "Guy Compounds"
  returns to the list (setActiveCompoundId(null)).
- Sidebar shows compound sub-items; clicking one sets activeCompoundId and
  scrollIntoView (though with drill-in this is scroll-to-top of detail).
- Footer nav steps through compounds in order, then advances to next section.

### Compound Detail View (CompoundDetailView component)
Three cards stacked:
  1. Measurements — 2-column grid (Anchor Radius ft, Relative Elevation ft), each with
     number input + flag button + check button.
  2. Guy Wires — blue-header table (see below).
  3. Photos — collapsible accordion with PhotoGroupRow per group.

### Wire Table (WireTable component)
Columns: Level | Pos. (L/R select) | Size (# Strands) | Strength Rating |
         Preform Color | Measured Tension (lbf) | Notes | Actions

- NOTE: Tension Propriety column was removed (not needed).
- overflow-x-auto scroll container; min-w-[640px] so it scrolls cleanly on small screens.
- Per-row actions: Flag (red) | Mark (indigo) | Remove (Minus → pending confirm).
- "Add Wire" footer button (indigo text).
- Empty state row with colspan.

### Deficiencies — Flat accordion (not drill-in)
All deficiencies live in one card (overflow-hidden) with divide-y between items.
Each DeficiencyAccordion:
  - Header: ChevronDown (rotates) + issue name + Flagged chip + photo count badge
  - Expanded fields rendered via DefFieldRow (3px red/green left bar + label + value box):
      Issue (select), Severity (select 1-Critical / 2-Moderate / 3-Low),
      Compounds Affected (LocationMultiSelect), Notes (textarea), Photo (thumbnail + link)
  - Flag + Mark Reviewed action buttons at bottom of each accordion
  - Remove button (Minus) on the accordion header
- Sidebar sub-items: clicking scrolls to that deficiency card via
  document.getElementById(`def_${id}`).scrollIntoView(...)

### Types
  GuyWire: id, guyLevel, position (L|R|''), size, strengthRating, preformColorCode,
           measuredTension, notes, flagged, marked
           (tensionPropriety field still in type but column removed from table)

  GuyCompound: id, label, anchorRadius, anchorRadiusFlagged, anchorRadiusMarked,
               relativeElevation, relativeElevationFlagged, relativeElevationMarked,
               photoGroups, wires

  GuySurvey: compounds (9: A B C AA BB CC AAA BBB CCC), guyPhotos, deficiencies, catchAll

### Mock Data
  Compound A: 4 wires (levels 1R/1L/2R/2L), anchorRadius=116, relativeElevation=0
  Compound B: 4 wires all GOOD, anchorRadius=116, relativeElevationFlagged=true
  C–CCC: empty
  4 deficiencies with severities 1-3

### Progress Helpers
  getCompoundProgress, getCompoundFlagCount, getSectionProgress,
  getSectionFlagCount, getOverallProgress — all wire into sidebar badges and
  toolbar progress bar.

### State
  activeCompoundId: string | null   — which compound detail is open
  activeDeficiencyId: string | null — which sidebar deficiency is highlighted
  activeSectionIdxRaw: number       — current top-level section index
  sidebarCollapsed, headerCollapsed, navSearch, rightTab, saveState, surveyComplete
